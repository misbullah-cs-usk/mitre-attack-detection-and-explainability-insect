## Running the SOC Dashboard

The `soc_dashboard` directory contains a FastAPI backend and a JavaScript frontend for demonstrating the MITRE ATT&CK prediction and LLM explanation pipeline.

The dashboard workflow is:

```text
Sysmon Event Input
        ↓
FastAPI Backend
        ↓
ML Hierarchical Classifier
        ↓
Ollama / Qwen LLM
        ↓
SOC Explanation Dashboard
```

---

## 1. Project Structure

Expected structure:

```text
├── soc_dashboard/
│   └── models/
│       ├── tfidf_sgd_parent.joblib
│       └── tfidf_sgd_subtechnique.joblib 
│   ├── app.py
│   ├── ml_predict_then_llm_explain.py
│   └── static/
│       ├── index.html
│       ├── app.js
│       └── style.css
```

The dashboard requires trained ML models:

```text
models/tfidf_sgd_parent.joblib
models/tfidf_sgd_subtechnique.joblib
```

These are generated by:

```bash
python 05_train_event_hierarchical_classical_ml.py
```

## 2. Install Required Packages

From the project root:

```bash
pip install fastapi uvicorn requests joblib numpy pandas scikit-learn
```

If you use a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate

pip install fastapi uvicorn requests joblib numpy pandas scikit-learn
```

---

## 3. Start Ollama

Make sure Ollama is running:

```bash
ollama serve
```

Check available models:

```bash
ollama list
```

Expected model:

```text
qwen2.5:3b-instruct
```

If the model is not available, pull it:

```bash
ollama pull qwen2.5:3b-instruct
```

---

## 4. Check LLM Model Name

Open:

```text
soc_dashboard/ml_predict_then_llm_explain.py
```

Make sure the model name matches your Ollama model:

```python
OLLAMA_MODEL = "qwen2.5:3b-instruct"
```

Also confirm the Ollama URL:

```python
OLLAMA_URL = "http://localhost:11434/api/generate"
```

---

## 5. Run the FastAPI Backend

From the project root, run:

```bash
cd soc_dashboard
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

If your `models/` directory is in the project root and not inside `soc_dashboard/`, run the server from the project root instead:

```bash
uvicorn soc_dashboard.app:app --host 0.0.0.0 --port 8000 --reload
```

Use whichever command matches your path setup.

---

## 6. Open the Dashboard

Open this URL in your browser:

```text
http://localhost:8000/static/index.html
```

If running on a remote server:

```text
http://<server-ip>:8000/static/index.html
```

---

## 7. Demo Workflow

1. Open the dashboard.
2. Enter or paste a Sysmon event.
3. Click **Analyze Alert**.
4. The dashboard will display:

   * Final ATT&CK prediction
   * Parent-technique or sub-technique level
   * Confidence score
   * Top parent-technique candidates
   * Top sub-technique candidates
   * LLM-generated SOC explanation
   * Recommended triage steps
   * Suggested Splunk searches

## 10. Troubleshooting

### Ollama connection error

Check that Ollama is running:

```bash
curl http://localhost:11434/api/tags
```

If it fails, start Ollama:

```bash
ollama serve
```

---

### Model file not found

Check that these files exist:

```bash
ls models/
```

Required files:

```text
tfidf_sgd_parent.joblib
tfidf_sgd_subtechnique.joblib
```

If missing, train them:

```bash
python 05_train_event_hierarchical_classical_ml.py
```

---

### Static files not loading

Make sure this directory exists:

```text
soc_dashboard/static/
```

Expected files:

```text
index.html
app.js
style.css
```

---

### Port already in use

Run with another port:

```bash
uvicorn soc_dashboard.app:app --host 0.0.0.0 --port 8080 --reload
```

Then open:

```text
http://localhost:8080/static/index.html
```
